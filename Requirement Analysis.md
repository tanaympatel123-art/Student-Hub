# Requirement Analysis

## Project Title

**StudentHub – Student Portal Website**

---

## Problem Definition

Students often have to visit multiple platforms to access attendance records, grades, subject information, and event announcements. Faculty members require a convenient way to manage academic records, while administrators need a centralized system to manage users and publish announcements.

StudentHub aims to provide a single web portal where students, faculty, and administrators can perform their respective tasks through dedicated dashboards.

---

## Project Scope

The project focuses on developing a responsive and accessible web portal with role-based navigation. Initially, the portal will be implemented using HTML5 semantic elements, with CSS and JavaScript added in later stages to improve design and functionality.

---

## Objectives

* Create a centralized student portal.
* Provide separate dashboards for Students, Faculty, and Admin.
* Display academic information in an organized manner.
* Improve accessibility using semantic HTML5.
* Maintain a structured project using Git and GitHub.

---

## User Roles

### Student

* Login to the portal
* View subjects
* Check attendance
* View grades
* Access event calendar
* Update personal settings

### Faculty

* Login to faculty dashboard
* Manage attendance
* Upload grades
* View academic information

### Admin

* Manage users
* Publish announcements
* Monitor portal activities
* Manage overall system settings
